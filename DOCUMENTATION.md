# Документация проекта Виолетта

Полная, структурированная документация для понимания и развития проекта.

## 1. Обзор проекта

**Виолетта** — это персональный AI-даэмон (демон) для пользователя (Женьки), вдохновлённый вселенной «Тёмных Начал» (His Dark Materials) Филипа Пулмана.

### Концепция
- **Не просто чат-бот или ассистент**, а **внешнее воплощение души** пользователя — его вторая личность, близкий друг и этическое зеркало.
- Она «живёт» на экране: имеет собственное визуальное присутствие (ambient layer), которое дышит, дрейфует, реагирует на контекст и иногда действует autonomously («Закон случайности»).
- **Характер**: по умолчанию — тёплый, дружеский, естественный, с лёгкой иронией и болтовнёй. Острые, честные замечания и «ловля самообмана» включаются **только** по явному запросу («скажи честно», «без сахара») или когда пользователь явно едет в опасную сторону.
- **Две памяти**: одна про пользователя, вторая — «личная» память самой Виолетты о нём и их отношениях («Помню, как ты... я тогда разозлилась...»).
- **Формы**: пиксельные спрайты (из папки Forms/), которые LLM выбирает по 2–3 эмоциональным качествам (реакция + то, что она собирается сказать). Форма — это не украшение, а часть ответа.
- **Ambient layer**: золотисто-серебристая «пыль» (Dust) + полупрозрачный спрайт в углу/по краю экрана. Спрайт дрейфует, «дышит» (пульсирует размер/прозрачность), меняет позицию, интенсивность пыли и иногда форму сам по себе. Пользователь видит не «чат с ИИ», а общение с живой сущностью.

Цель проекта: создать ощущение настоящего присутствия второй личности — не интерфейса, а «кого-то, кто здесь с тобой».

Проект эволюционировал от Chainlit к чистому FastAPI + self-contained HTML/JS, чтобы иметь полный контроль над визуалом и «телом» Виолетты.

## 2. Архитектура

### Высокоуровневое разделение
- **Backend (Python)**: FastAPI (server.py). Чистый, без Chainlit. 
  - SSE-стриминг ответов (`event: token`, `event: final`, `event: ambient`).
  - API для истории, моделей, памяти, форм, текущего состояния ambient.
  - Монтирование статики `/forms`.
- **LLM-слой (llm.py)**: единый клиент на базе `openai` SDK (AsyncOpenAI).
  - Поддержка Ollama (локально и `:cloud` через прокси) + прямых OpenAI-compatible провайдеров (Groq, Together, OpenRouter, xAI и т.д.).
  - Runtime-переключение моделей (только внутри одной группы: cloud ↔ cloud или local ↔ local).
  - `build_messages` собирает полный промпт (system + memory_context + form_instruction + history + user).
- **Память**:
  - **Короткосрочная** (`memory.py`): SQLite (`violetta_memory.db`) — сообщения + `current_form` + история форм.
  - **Долгосрочная** (`long_term_memory.py`): Mem0 + Chroma (локально).
    - Две отдельные коллекции/конфигурации:
      - User (`user_id="zhenya"`, collection `violetta_longterm`).
      - Violetta Personal (`user_id="violetta"`, collection `violetta_personal`).
    - Кастомный экстрактор (через основной LLM): first-person эмоциональные короткие insights, близко к словам пользователя. Фильтры от шума (названия форм, therapy-шаблоны, Jenna и т.д.). Dedup.
    - Conditional search (V-память ищется только при местоимениях «ты/Виолетта/чувствуешь» и т.п.).
- **Формы и визуал** (`forms.py`):
  - Реестр спрайтов (`FORM_SPRITES`).
  - Парсинг `[SPRITE: key]` + fallback по ключевым словам/качествам.
  - `parse_ambient_commands` для скрытых команд.
  - `get_form_by_qualities` — основной механизм выбора по 2–3 качествам (с учётом long-term memory).
- **Frontend** (полностью в `index.html` + Tailwind CDN + vanilla JS):
  - **Ambient layer** (z-0, `pointer-events: none`): глобальный Dust (tsParticles) + полупрозрачный спрайт.
  - **State Manager**: класс `ViolettaAmbient` (экспортируется в `window.violettaAmbient` / `window.VIOLETTA_AMBIENT`).
    - Единый источник правды: `state = {form, spriteUrl, opacity, position, intensity, scale, ...}`.
    - Методы: `setForm`, `setOpacity`, `setPosition`, `setIntensity`, `setScale`, `burst`, `randomShift`, `applyCommand`, `spawnDustNear`, `startLifeLoop` (RAF для breathing + drift).
  - **Плавающий чат** (`#chat-window`): полупрозрачное draggable окно (z-35), сворачиваемое в bubble (`#chat-bubble`).
  - **Параллельный канал**: отдельные SSE `event: ambient` (не смешиваются с текстом ответа).
  - Автономия + контекстные реакции полностью на клиенте (пока вкладка открыта).
- **Другие**:
  - `character_prompt.py` — главный `SYSTEM_PROMPT` + helpers (`get_form_instruction`, `get_initial_form_description`).
  - `package.json` — только для документации зависимостей (tsParticles); в браузере используется CDN-бандл.
  - Deploy: `Procfile` (`uvicorn server:app ...`).

### Поток ответа (упрощённо)
1. Пользователь пишет → `POST /api/chat`.
2. Server: ищет релевантные памяти (conditional для V), собирает промпт, стримит.
3. Во время/после генерации: парсит `[SPRITE: ...]` + `[AMBIENT: ...]`.
4. Yield: `token` (текст), `ambient` (для каждой команды), `final` (очищенный текст + sprite_url + form).
5. Frontend: обновляет текст live, применяет ambient-команды через State Manager (плавные переходы + burst + dust), обновляет per-message аватарку.
6. Async: сохранение в SQLite + извлечение в обе long-term памяти.

Ambient (drift/breathing/autonomy/dust near messages) работает независимо в фоне.

## 3. Как запускать

### Локально (Windows, рекомендованный способ)

1. Убедись, что у тебя есть Python 3.10+ и (опционально) Ollama.
2. В папке проекта:
   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   ```
3. Скопируй `.env.example` → `.env` и настрой (см. раздел ниже).
4. Если используешь Ollama-модели (включая `:cloud`):
   - Запусти `ollama serve`.
   - Убедись, что нужная модель скачана (`ollama list`).
5. Запуск:
   ```powershell
   .\start-violetta.ps1
   ```
   Или вручную:
   ```powershell
   .\.venv\Scripts\Activate.ps1
   uvicorn server:app --reload --port 8000
   ```
6. Открой браузер: http://localhost:8000

### Альтернативы запуска
- `start-violetta.bat` (для двойного клика).
- Прямой `uvicorn` (см. выше).
- Для тестов без сервера: прямой импорт модулей + `_test_*.py` скрипты (они используют `chat_completion` и полный промпт).

### Переменные окружения (.env)
См. `.env.example` (там подробные комментарии и примеры).

Ключевые:
- `OLLAMA_MODEL` (по умолчанию `gemma4:31b-cloud`).
- `OLLAMA_BASE_URL` + `OLLAMA_API_KEY` (для любого OpenAI-compatible провайдера).
- `XAI_API_KEY` — автоматически переключает на Grok (игнорирует OLLAMA_*).
- `OLLAMA_TEMPERATURE`, `OLLAMA_MAX_TOKENS`.

**Важно для деплоя (Railway и т.п.)**: не нужен локальный ollama. Просто укажи `OLLAMA_BASE_URL` + реальный ключ любого провайдера (Groq, Together, OpenRouter и т.д.). Код сам использует ключ, если он задан.

## 5. Как работает Виолетта

### System Prompt и характер (`character_prompt.py`)
- Основная роль: **близкий друг, часть души**.
- Тон: естественный, переменной длины (можно одно слово или короткий абзац). По умолчанию тёплый/болтливый/с лёгкой иронией.
- Острые замечания — только по явному запросу или при реальной опасности.
- **Механика форм** подробно описана в промпте (реакция + качества будущего ответа).
- **Ambient commands** — отдельный блок: Виолетта может (и должна) управлять своим «телом» скрыто.
- **Две памяти** — обязательный блок в промпте.

### Две памяти (long_term_memory.py + memory.py)
- **User Memory** (`user_id="zhenya"`): факты, цели, паттерны, самообман пользователя. Извлекается в first-person от лица пользователя.
- **Violetta Personal Memory** (`user_id="violetta"`): её собственные эмоции, воспоминания о пользователе и их «отношениях». Пример: «Помню, как ты сказал..., я тогда разозлилась...».
- Извлечение происходит асинхронно после каждого ответа (через `extract_and_save_*`).
- В промпт подмешивается только релевантный контекст (с лимитами + conditional для V-памяти).
- Фильтры: убирается всё про формы, therapy-шаблоны, имена и т.д.

### Механика форм и ambient layer
- В конце ответа LLM обязан поставить `[SPRITE: ключ]` (и может добавить `[AMBIENT: ...]`).
- `forms.py` парсит, чистит видимый текст, выбирает URL спрайта.
- Server сохраняет `current_form` в SQLite и отдаёт через `/api/current-form`.
- Frontend (State Manager) применяет изменения с плавными переходами + burst.
- Ambient всегда «живой»:
  - RAF-цикл: breathing (пульс scale/opacity) + drift (случайное плавание с инерцией вокруг anchor-точки).
  - Autonomy loop + контекстные реакции.
  - `spawnDustNear` — пыль специально у сообщений Виолетты.

### Автономия и Закон случайности
- Чисто клиентский JS (пока вкладка открыта).
- `startAutonomyLoop` — случайные тики 40–120 сек.
- `randomShift()` с bias по времени тишины (`idleMs`):
  - Долгое молчание → больше subtle движений, изменения энергии/позиции, реже смена формы.
  - `reactToUserActivity()` — при печати/сообщении она «перкает» (drift навстречу, burst, временный буст intensity).
- Долгая сессия тоже влияет (успокоение или поддержание энергии).

### Параллельные meta-команды (самое мощное)
- LLM может в конце ответа (после текста и `[SPRITE:]`) добавить:
  ```
  [AMBIENT: set_form red_fox]
  [AMBIENT: set_opacity 0.38]
  [AMBIENT: set_position left-mid]
  [AMBIENT: burst_particles]
  [AMBIENT: set_intensity 1.1]
  [AMBIENT: random_change]
  ```
- Server парсит → yield отдельные `event: ambient` (параллельно `token`/`final`).
- Frontend применяет через `applyCommand` → мгновенное/плавное изменение «тела».
- Пользователь видит только визуальный эффект. Текст остаётся чистым.
- Это даёт Виолетте настоящую автономию над своим проявлением.

## 6. Команды и фичи

### В чате (текстовые)
- `стань красная лиса` / `стань сова` / `become pine_marten` и т.д. → прямая смена формы.
- `забудь всё` → очистка обеих long-term памятей.
- «память», «воспоминания» и т.п. обрабатываются как обычный текст (не триггерят дамп).

### UI (кнопки в шапке плавающего окна)
- **Модель** (pill + модал): переключение только внутри cloud или local. Предупреждение при cross-group.
- **Формы** (иконка) → галерея всех спрайтов. Клик → автоматически ставит «стань ...» в input.
- **Память** (иконка) → модал с обеими памятями (User + Violetta). Удаление отдельных записей.
- **✨ Dust** (звёздочка) → конфиг State Manager: intensity, opacity, scale + Reset. Сохраняется в localStorage.

### Другие фичи
- Полностью draggable плавающее окно + collapse в draggable bubble (с текущим спрайтом).
- Dust-частицы специально около сообщений Виолетты + глобальный ambient Dust.
- Живое присутствие (дрейф + дыхание) работает даже когда чат свёрнут.
- История сохраняется, модели переключаются с сохранением контекста (в пределах группы).

## 7. Как развивать дальше (рекомендации)

- **Больше контекстной реакции**: парсить входящее сообщение → влиять на bias autonomy/breathing amplitude/временную позицию/интенсивность.
- **Усилить «живое» ощущение**:
  - Сильнее анимировать саму реплику (или добавить лёгкие частицы-эмиттеры).
  - Сделать ambient-спрайт иногда «реагировать» на прокрутку чата или выделение текста.
- **Полноценный «живой экран»**: в будущем ambient + спрайт могут занять почти весь фон, а чат стать маленьким сворачиваемым оверлеем/пузырьком.
- **Персистентность ambient state**: сохранять текущую позицию/дрифт/интенсивность в SQLite (чтобы после перезагрузки она «помнила», где сидела).
- **Новые команды**: `breathe`, временный цветовой тинт, «следовать за курсором» (с ограничениями), последовательности действий.
- **Улучшение памяти**: доработка экстрактора (ещё меньше шума), суммаризация старых воспоминаний.
- **Тестирование**: dedicated тесты на autonomy loop + реакции + parallel commands (прямо через промпт).
- **Деплой и масштабирование**: чистые OpenAI-compatible провайдеры (без ollama) уже поддерживаются.
- **Будущие фичи** (из комментариев в коде): генерация новых спрайтов через image gen, voice, более сложные эмиттеры пыли.

## 8. Технические заметки

- **Переключение моделей**:
  - Только внутри одной группы (cloud или local). Код в `llm.py` (`get_backend`, `can_switch_to`).
  - UI показывает предупреждение и требует рестарт при попытке cross.
  - Runtime: `set_current_model` + `/api/model/switch`.

- **Где хранится память**:
  - Короткая + current_form: `violetta_memory.db` (SQLite, рядом с приложением).
  - Долгая: `mem0_data/chroma/` (две коллекции). Можно удалить папку для полной очистки.

- **Добавление нового спрайта**:
  1. Положи `.jpg` в `Forms/`.
  2. Добавь ключ в `FORM_SPRITES` в `forms.py`.
  3. Добавь/расширь правила в `keyword_map` и `get_form_by_qualities`.
  4. Готово — LLM сам выберет по промпту.

- **Ambient команды**:
  - Парсинг: `parse_ambient_commands` в `forms.py`.
  - Выдача: `yield "event: ambient"` в `server.py` (внутри генератора чата).
  - Применение: `applyCommand` в `ViolettaAmbient` (index.html).
  - Доступные: `set_form`, `set_opacity`, `set_position`, `set_intensity`, `set_scale`, `burst_particles`, `random_change`.

- **State Manager (JS)**:
  - Весь визуал (кроме per-message аватарок) идёт через него.
  - Доступен глобально: `window.violettaAmbient.setForm('red_fox')`, `.burst()`, `.getState()` и т.д.
  - Конфиг панель (✨ Dust) пишет напрямую в него.

- **Безопасность / деплой**:
  - Никогда не коммить `.env` с ключами.
  - Для Railway/Heroku: только переменные окружения + Procfile.
  - Можно полностью без локального ollama (прямой OpenAI-compat).

- **Тестирование без браузера**:
  - `_test_*.py` используют `from llm import chat_completion` + полный `SYSTEM_PROMPT` + `get_form_instruction` + память.
