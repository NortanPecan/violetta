# Виолетта — честный AI-даэмон

**Внешнее воплощение души.** Радикально честный спутник, зеркало и вторая личность. Фокус на самообмане, глубокой рефлексии, целях и внутреннем росте. Без сахара, без терапии, без подлизывания.

## Быстрый старт (если не хочешь читать полную доку сразу)

Текущая реализация: чистый FastAPI + Tailwind HTML/JS (без Chainlit). Полноценный SPA с аватаркой слева, естественным чатом, переключателем моделей, галереей форм и памятью.

## Ключевые фичи

- **Две памяти (Mem0 + Chroma)**:
  - User Memory: про тебя (цели, паттерны, самообман, события).
  - Violetta Personal Memory: мои собственные эмоции, отношение к тебе, воспоминания о наших разговорах с моей точки зрения. Я могу ссылаться на них естественно ("Помню, как ты... я тогда разозлилась...").
- **Fluid пиксельные формы**: LLM выбирает спрайт по 2-3 качествам (твоя эмоция + моя реакция + что я собираюсь сказать). Форма — часть ответа визуально.
- **Умный переключатель моделей** в UI (клик по кнопке сверху): отдельные списки Cloud и Local. Переключение только внутри группы. При попытке cross — предупреждение + требование рестарта.
- Поддержка Ollama (локальные + :cloud модели через proxy) и xAI/Grok (если XAI_API_KEY).
- Естественные короткие/острые ответы (2-5 предложений в среднем), сарказм, первый человек в памяти.
- Полная история в SQLite, формы эволюционируют.
- Всё в одном репо, легко деплоить.
- **Фоновое визуальное присутствие + автономия**: tsParticles Dust (золотисто-серебристая пыль в стиле Тёмных Начал). Спрайт-присутствие в углу с плавными переходами и burst'ами. 
  - Параллельный канал: LLM может отдавать скрытые `[AMBIENT: set_form red_fox]`, `set_opacity`, `set_position`, `burst_particles`, `set_intensity`, `random_change` и т.д. (отдельно от текста и [SPRITE:]). 
  - State Manager в JS (централизованный).
  - Автономия: каждые ~40-120с с небольшой вероятностью Виолетта *сама* меняет форму/позицию/opacity/intensity/делает burst — просто потому что "ей захотелось". Работает, даже если ты молчишь (пока вкладка открыта).
  - Конфиг: кнопка ✨ Dust (слайдеры + reset). Всё через manager, легко расширять.
  - Живое присутствие: постоянное дыхание (пульс размера/прозрачности), естественный drift (плавает в радиусе ~80px вокруг опорной точки), контекстные реакции (при твоём сообщении/печатании — "наклоняется", burst, больше пыли), умная автономия при долгом молчании (чаще subtle движения/изменения энергии, реже резкие формы). Ощущение живого существа в углу, а не картинки.

## Быстрый запуск локально (Windows)

1. Убедись, что Ollama запущен (`ollama serve`).
2. `cd projects/violetta`
3. `.\start-violetta.ps1` (или .bat)
4. Откроется http://localhost:8000

Или вручную:
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn server:app --reload --port 8000
```

Скопируй `.env.example` в `.env` и настрой модель/ключи.

## Переменные окружения (.env)

См. `.env.example`

Ключевые:
- `OLLAMA_MODEL=gemma4:31b-cloud` (или локальная gemma4:e4b и т.д.)
- `OLLAMA_BASE_URL=http://localhost:11434/v1`
- `OLLAMA_API_KEY=...` (для paid cloud моделей)
- `XAI_API_KEY=...` — если установлен, переключается на Grok/xAI (полезно для деплоя)

## Деплой на Railway

1. Загрузи этот репозиторий на GitHub (как ты просил).
2. Зайди на [Railway.app](https://railway.app), New Project → Deploy from GitHub repo.
3. Выбери репозиторий `violetta`.
4. Railway автоматически соберёт Python + увидит Procfile.
5. **Обязательно добавь Variables** (в Dashboard проекта):
   - `OLLAMA_MODEL` (рекомендую `gemma4:31b-cloud` или переключи на Grok)
   - Для облачных моделей Ollama: на Railway нет локального ollama. 
     - Лучший вариант для деплоя: установи `XAI_API_KEY` (твой ключ xAI) — код автоматически переключится на Grok модели через `api.x.ai`.
     - Альтернатива: разверни отдельный сервис Ollama на Railway (есть шаблоны) и укажи его URL в `OLLAMA_BASE_URL`.
   - Другие: `OLLAMA_TEMPERATURE=0.6`, `OLLAMA_MAX_TOKENS=380`
6. Deploy. Приложение будет на случайном URL Railway.

**Примечание по моделям на Railway / любом хостинге без ollama:**
- Текущие `*:cloud` модели (gemma4:31b-cloud и т.п.) работают **только через локальный `ollama serve`** как прокси. На Railway без ollama они не заработают.
- **Рекомендуемый вариант для деплоя:** используй прямые OpenAI-compatible провайдеры.
  - Поставь `OLLAMA_BASE_URL` и `OLLAMA_API_KEY` на любой провайдер (Groq, Together.ai, OpenRouter, Fireworks и т.д.).
  - Код теперь автоматически использует настоящий ключ, если он задан (см. обновлённый `.env.example` с примерами).
- Альтернатива: `XAI_API_KEY` → автоматически уйдёт на Grok модели через xAI.
- Полный пример в `.env.example`.

Это позволяет запускать Виолетту на Railway без установки ollama вообще.

## Структура

- `server.py` — FastAPI бэкенд + SSE для стриминга
- `index.html` + Tailwind — чистый фронтенд (аватар слева, чат справа, модалы для форм и памяти)
- `llm.py` — абстракция над OpenAI-совместимыми (Ollama / xAI)
- `character_prompt.py` — главный System Prompt (две памяти + механика форм)
- `long_term_memory.py` — Mem0 + две памяти + пост-обработка в первый человек
- `forms.py` — выбор спрайтов по качествам
- `memory.py` — SQLite для короткой истории и текущей формы
- `Forms/` — твои 15+ пиксельных спрайтов

## Добавление новой формы

1. Положи .jpg в `Forms/`
2. Добавь в `forms.py` в `FORM_SPRITES` и `RUSSIAN_KEYWORDS`
3. Готово (LLM сам выберет по промпту)

## Фоновое присутствие (ambient layer)

- Частицы: лёгкая золотисто-серебристая пыль (Dust). Полупрозрачные, медленно плавают, с лёгким мерцанием.
- Спрайт формы: в правом нижнем углу, ~168px, opacity ~0.22 по умолчанию, round, с мягким золотым glow. Сидит поверх частиц, под чатом.
- Смена: при ответе с новой формой — плавный fade + scale + всплеск частиц вокруг спрайта.
- Конфиг: кнопка со звёздочкой в шапке (Dust). Слайдеры + Apply/Reset. Значения в localStorage.
- Техника: CDN bundle tsparticles (v3), npm пакеты установлены (package.json) для будущего. Полностью pointer-events: none, не мешает вводу.
- Чтобы поменять позицию/стили: редактируй #ambient-sprite и .ambient-vignette в index.html (CSS vars можно вынести позже).

Запуск тот же. Работает в Railway (CDN, без node на рантайме).
